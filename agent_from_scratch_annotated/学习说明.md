## 先读哪个文件

建议顺序：

1. `single_agent_example.py`
   - 先看最外层程序怎么启动。
   - 看 `messages = []` 为什么要一直保存。
   - 看用户输入如何进入 `client.run(...)`。

2. `agent.py` 里的 `Swarm.run()`
   - 这是 Agent loop 主体。
   - 重点看：模型调用、assistant 消息、tool_calls、工具结果回填、停止条件。

3. `agent.py` 里的 `handle_tool_calls()`
   - 重点看：工具名如何映射到 Python 函数。
   - 看 `json.loads(arguments)` 和 `function_map[name](**args)`。

4. `agent.py` 里的 `function_to_json()`
   - 看普通 Python 函数如何变成工具 schema。
   - 对照你现在手写的 `tool_registry.py`。

5. `multi_agent_example.py`
   - 最后看多 Agent 转交。
   - 暂时只理解“工具返回 Agent 可以切换 active_agent”，不用急着实现。

## 和你 day11 当前代码的对应关系

| 这个仓库 | 作用 | 你 day11 项目里的对应概念 |
|---|---|---|
| `Agent(name, model, instructions, functions)` | 保存 Agent 配置 | 你现在还没有集中式 Agent 类，配置分散在 `model.py` / `tool_registry.py` |
| `function_to_json()` | 自动生成工具 schema | `tool_registry.py` 现在是手写工具说明 |
| `Swarm.get_chat_completion()` | 带着 messages + tools 调模型 | `model.py` |
| `Swarm.handle_tool_calls()` | 执行模型请求的工具 | `run_tool.py`，但这里没有你单独的 parser 层 |
| `Swarm.run()` | Agent 主循环 | `main.py` / `run_agent()` |
| `messages` | 会话轨迹 | `session.json` / memory 里的消息记录 |
| `Result` | 统一包装工具返回值 | 你当前工具返回的 `{"success", "message", "data"}` |
| 工具返回 `Agent` | 多 Agent 转交 | 暂时属于后续阅读理解，不急着接入 |

## 读 `Swarm.run()` 时抓这条主线

```text
active_agent = agent
history = copy.deepcopy(messages)

while 没到 max_turns:
    completion = 调模型(active_agent, history)
    message = 模型返回的 assistant 消息
    history.append(message)

    if 没有 tool_calls:
        break

    partial_response = 执行工具(tool_calls)
    history.extend(tool 消息)

    if 工具返回了新 Agent:
        active_agent = 新 Agent

return 本次新增 messages + 当前 active_agent
```

这就是 Agent loop 的核心。

## 读 `handle_tool_calls()` 时抓这条主线

```text
function_map = {函数名: 函数对象}

for tool_call in tool_calls:
    name = 模型要调用的工具名
    args = json.loads(模型给的 arguments)
    raw_result = function_map[name](**args)
    result = 统一包装工具结果
    生成 role=tool 的消息
```

对照你当前项目就是：

```text
模型输出 tool_name + arguments
↓
parse_tool_call_result.py 校验
↓
run_tool.py 分发
↓
tools/*.py 执行
↓
工具结果写回 messages/session
```

## 你现在可以吸收的设计

1. `messages` 是核心轨迹，不是临时变量。
2. 工具结果必须变成 `role=tool` 的消息，再放回模型上下文。
3. 工具注册可以从普通函数自动生成 schema。
4. `active_agent` 是多 Agent 转交的核心状态。
5. Agent loop 的停止条件通常是：模型不再请求工具，或者达到最大轮数。

## 你现在先不要急着学的设计

1. 不急着把你的 `tool_registry.py` 改成自动 inspect 生成。
2. 不急着实现多 Agent handoff。
3. 不急着支持 parallel tool calls。
4. 不急着照搬 Pydantic 容器。
5. 不急着完全改成 OpenAI tools API；你现在本地模型 JSON 工具调用路线依然适合练基础。

## 重点提醒

这个仓库依赖 OpenAI API，不一定适合你现在直接跑。
你当前的目标不是“马上运行它”，而是读懂它的结构：

```text
模型决策 → 工具调用 → 工具结果放回 messages → 下一轮模型继续判断
```

如果你能用自己的话讲清楚这条链路，再回头看自己的 day11，结构意识会明显增强。
